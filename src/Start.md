# 开始你的第一个 DRG 模组

作为示例，我将简单制作一个玩家瞬移的模组，没有太多要求，仅仅是将玩家往视线方向传送一定距离。

打开项目，在内容（Content）下新建一个文件夹，你可以任意命名，这是存放你模组内容的地方，这里我命名为 `_Tutorial` 。

![1-1](images/1/1.png)

右键空白区域新建蓝图类，选择 Actor。

![1-2](images/1/2.png)

名字这里我取PlayerTP，可以是任意，只要你能记住。

![1-3](images/1/3.png)

然后双击该蓝图，就会打开称之为编辑器的窗口。

![1-4](images/1/4.png)

我们大部分的操作基本上都会在这个编辑器里进行。
在这一章里，你不需要了解太多东西，只需要跟着教程的指示，目的是快速了解蓝图 Mod 的制作流程。

在类默认选项中，取消勾选启用 Tick 和可被伤害。

![1-5](images/1/5.png)

然后切换到事件图表。
可以看到默认有三个事件，通常情况下只会用到第一个，所以后面两个可以删掉。

![1-6](images/1/6.png)

Event BeginPlay 事件会在 Actor 实例化时立即执行，Mod 的逻辑也是从这里开始。
右击编辑器空白处，搜索 `GetLoaclPlayerCharacter` 函数。

![1-7](images/1/7.png)
![1-8](images/1/8.png)

这是游戏提供的函数，可以快速得到本地玩家的引用。
然后在该函数的输出 `ReturnValue` 按住左键拖出，继续输入 `IsValid`。

![1-9](images/1/9.png)
![1-10](images/1/10.png)

这一步用于判断是否获得有效的引用，因为如果当前游戏场景处于加载状态或玩家未存在于场景中，只会得到一个无效的引用，一定要确保不能出现空引用的情况。
在 `IsNotValid` 的执行引脚处按住左键拖出，输入 `Delay`。并按照下图连接。

![1-11](images/1/11.png)

这么做是为了如果得到了无效引用，就延迟0.2s后继续获取，直至得到有效值。
你可以双击执行引脚添加一个路由点，让你的蓝图保持清晰可读。

![1-12](images/1/12.png)

到这里相信你已经明白各种执行线的连接方式。
继续在 `IsValid` 的引脚处执行 `EnableInput`，为该蓝图获取玩家的输入功能。

![1-13](images/1/13.png)

从本地玩家的引用中获取玩家控制器连接到启用输入函数的输入。

![1-14](images/1/14.png)

看起来就像这样。

我们希望当玩家按下某个快捷键时向摄像机视角的方向瞬移一段距离。
在编辑器空白处右键搜索 `Key`。

![1-15](images/1/15.png)

找到你喜欢的按键，我这里使用 T 键作为演示。
单击这个事件，在右边取消勾选消耗输入和重载父类绑定。

![1-16](images/1/16.png)

接着获取玩家当前的位置和角度。

![1-17](images/1/17.png)

这样就能得到玩家视线的方向。

![1-18](images/1/18.png)

将这个向量与浮点相乘。

![1-19](images/1/19.png)

这个浮点值就是我们希望瞬移的距离，在引擎中，单位通常是cm，即100就是1米。

你可以右击这个绿色的浮点输入，将它提升为变量。

![1-20](images/1/20.png)

将变量命名并填入你希望瞬移的距离。

![1-21](images/1/21.png)

然后将玩家当前的位置向量与瞬移的向量长度相加，得到的就是玩家瞬移之后的正确位置。
这时候只需要更新玩家位置就实现了瞬移效果。

最终的蓝图连接方式。

![1-22](images/1/22.png)

这时候可以松一口气，关掉编辑器窗口，但还有最重要的工作没有完成。

想让你的蓝图能够正确加载到场景，还需要两个 Actor，`InitSpacerig` `InitCave`。

这两个分别对应太空站台和洞穴，当游戏处于太空站台或洞穴时，仅会加载对应的Init。
你可以直接在这两个 Actor 内 Spawn 你的 Mod 蓝图，但是也有更简单的方法。

右击你的蓝图类创建两个子蓝图类，分别改成对应的名字。
原因是当一个 Actor 被加载时，它所继承的父类也必然会被加载，你的主蓝图就会以这种方式加载到场景中。

![1-23](images/1/23.png)
![1-24](images/1/24.png)

最后只需要打包成 pak，Mod 就完成了。
